Hello World! I made this blog to show off some of the projects I work on. It seems like most places are asking for a website when you apply to them nowadays, so I will use this one. Hopefully this website represents what I have done well. I am going to do my best to have it do that.

I figured that after learning how to be a full stack developer while I was employed at Stanford, I might as well put the knowledge to good use and make a webapp. While I was at Stanford, I programmed a webapp that ran on an Apache server using the Jinja2 template engine, mod_wsgi, and Google's webapp2. That webapp made use of a Postgresql database to store information about zones, vsyses, ip addresses, ports, and users pertaining to Stanford's network. One webapp I programmed routinely updates Stanford's border routers to prevent malicious traffic from being on the Stanford net. Another webapp I wrote allowed project managers to modify the firewall rules on Stanford's network that pertained to their project. Between those two projects I learned how to use HTML, CSS, Javascript, Python, Postgresql CLI commands, and exaBGP all in the same summer. I am incredibly grateful for the opprotunity I had at Stanford, and the amazing amount of information they taught me about computer networks and systems.

So the first project I am going to display is this webapp. The web page you are looking at now was served to you by an Ubuntu server running apache2 which makes use of mod_wsgi. Django is the web framework I used, which had built in support for Jinja2, and Postgresql (even though I use psycopg2 directly anyway). I use the database to store and retrieve blog entries. I originally styled the site with Bootstrap because of a class I had my senior year in college. I have since changed to use w3.css because it is lighter, looks a lot like Bootstrap, and, in my opinion, is simpler. The dark style of the site I wrote myself because it is easier on the eyes than a white screen with black text.

Using mod_wsgi in daemon mode would make some people say that this is a production grade webserver, and Django with Jinja2 is quite a lot more versatile than typical static pages that use Javascript to appear 'dynamic'. I suppose Ajax may be able to do what I am using Jinja2 for, but I like keeping the code that interfaces with the database serverside. It wasn't too hard to set all this up. All it required was reading documentaion. To give you an idea of how much work it was, or if you would like to do it yourself, follow these steps below.

  1. Get a computer that your willing to leave on 24/7 hooked up to the internet, and install Ubuntu on it. I did this with Ubuntu 16.04.1, but it will probably work on other versions of Ubuntu. Read how to install Ubuntu here.
  2. Make sure it is connected to the internet
  3. Ensure Python is installed. I did it with Python 3, but I know 2.7 is able to do it because that is the version of Python the Stanford servers used. I am pretty sure Ubuntu comes with python already installed. You can check by typing

    python --version

    on the command line. If it has a number denoting a version, then python is installed. If not, you can install it on the command line with

    sudo apt-get install python3

  4. Install pip with

    sudo apt-get install python3-pip

  5. Install mod_wsgi with

    sudo pip install mod_wsgi

  6. Install Django with

    sudo pip install Django==1.10.4

  7. Configure mod_wsgi with Django by following the guidelines found here. Also don't forget to set it up to use Daemon mode because people on the internet say that mode is better for memory. Information on that can be found here. In addition, Django provides some helpful information about what to have in the configuration files. That can be found here. This is by far the hardest part because there are so many configuration files and so many different ways of configuring that work. They all have tradeoffs. Good luck.
  8. Once that is done, the site is basically ready for development. Information on starting a Django project can be found here. Django's site also provides tutorials on how to setup and use databases, such as Postgresql, and how to use templating engines, like Jinja2. I followed those tutorials to make this webapp.
  9. Set Django to be able to serve static files so that CSS and Javascript are supported. Information on how to do this can be found here.

After following all those tutorials, the development environment is built. That is not all that is required though. Django requires that the name that is the url for the page is registered in settings.py, port forwarding needs to be setup so that http requests to the IP address the server is on are handled by the server, and a domain name must be registered. Django should probably be changed to not be in debug mode, but I don't think it is a security risk so I left it. You can check out what happens with debug mode by entering a url for this website that is not valid, such as ryanloringcooper.info/root or something like that. If there is some security risk by doing that, please let me know by emailing me at my address found on thecontact page.

I made a git repo to keep track of my changes to this site. It has all the files that are used for this webapp. You can check it out here.
